## 操作步骤

### 📤 上传ZIP文件

#### 方式1: 拖拽上传
1. 将zip文件拖动到上传区域
2. 松开鼠标，文件信息会显示出来
3. （可选）在输入框中输入自定义项目名称
4. 点击"上传并解压"按钮

#### 方式2: 点击选择
1. 点击上传区域
2. 在文件选择对话框中选择zip文件
3. （可选）输入自定义项目名称
4. 点击"上传并解压"按钮

### 📝 项目命名规则

- **不输入项目名**: 使用zip文件名作为项目名（去掉.zip后缀）
  - 例如: `my-project.zip` → 项目名为 `my-project`

- **输入项目名**: 使用自定义名称
  - 例如: 输入 `demo` → 项目名为 `demo`

- **同名项目处理**: 自动备份原项目
  - 原项目: `my-project`
  - 备份为: `my-project_backup_20260210_123456`

### 🔍 查看项目

#### 项目卡片信息
每个项目显示：
- 项目名称
- 最后修改时间
- 访问按钮
- 删除按钮

#### 访问项目
1. 在项目列表中找到目标项目
2. 点击"🔗 访问"按钮
3. 在新标签页中打开项目

或直接在浏览器输入:
```
http://your-server-ip/项目名称/
```

### 🗑️ 删除项目

1. 在项目列表中找到要删除的项目
2. 点击"🗑️ 删除"按钮
3. 在确认对话框中点击"确定"
4. 项目将被永久删除（不可恢复）

### 🔄 刷新列表

点击右上角的"🔄 刷新"按钮更新项目列表

## 上传状态说明

### 状态显示

#### ⏳ 上传中
- 显示进度条
- 显示上传百分比
- 上传按钮禁用

#### ✅ 上传成功
- 绿色提示框
- 显示项目名称和访问链接
- 如有备份，显示备份信息
- 2秒后自动重置表单

#### ❌ 上传失败
- 红色提示框
- 显示错误原因
- 可重新上传

## ZIP文件要求

### ✅ 正确的ZIP结构

```
my-project.zip
└── my-project/
    ├── index.html
    ├── css/
    │   └── style.css
    ├── js/
    │   └── script.js
    └── images/
        └── logo.png
```

或者直接包含文件（不推荐，但也可以）:
```
my-project.zip
├── index.html
├── css/
├── js/
└── images/
```

### ❌ 不支持的格式

- .rar 文件
- .7z 文件  
- .tar.gz 文件
- 未压缩的文件夹

### 文件限制

- **格式**: 仅支持 .zip
- **大小**: 最大 100MB（可在配置中修改）
- **编码**: 支持中文文件名（UTF-8）

## 最佳实践

### 📦 ZIP文件准备

1. **确保入口文件**
   - 项目根目录应包含 `index.html`
   - 或在子目录中包含首页文件

2. **优化文件结构**
   ```
   project/
   ├── index.html          (必需)
   ├── css/
   ├── js/
   ├── images/
   └── fonts/
   ```

3. **检查文件大小**
   - 压缩前检查项目大小
   - 移除不必要的源文件和缓存
   - 优化图片资源

4. **使用相对路径**
   - CSS: `<link href="css/style.css">`
   - JS: `<script src="js/app.js">`
   - 图片: `<img src="images/logo.png">`

### 🎯 命名建议

- 使用英文和数字
- 使用连字符分隔: `my-project`
- 避免空格和特殊字符
- 保持简短有意义

### ⚡ 效率提升

1. **批量上传**: 先准备好多个zip文件，依次上传
2. **版本管理**: 使用 `v1`, `v2` 等后缀区分版本
3. **定期清理**: 删除不再使用的旧项目
4. **备份重要**: 上传前保留本地备份

## 常见使用场景

### 场景1: 前端开发原型预览
```
1. 在本地完成HTML原型开发
2. 压缩为zip文件
3. 上传到服务器
4. 分享URL给团队成员查看
```

### 场景2: 客户演示
```
1. 准备演示版本
2. 上传为 `client-demo`
3. 生成访问链接
4. 发送给客户预览
```

### 场景3: 多版本管理
```
1. 上传 `project-v1`
2. 客户反馈后上传 `project-v2`  
3. 保留多版本供对比
4. 确认后删除旧版本
```

### 场景4: 快速迭代
```
1. 上传初版 `prototype`
2. 修改后重新上传（自动备份旧版）
3. 对比新旧版本
4. 确认无误删除备份
```

## 提示与技巧

💡 **提示1**: 上传前可以在项目名称输入框中使用有意义的名称

💡 **提示2**: 同名项目会自动备份，不用担心覆盖

💡 **提示3**: 备份的项目也会显示在列表中，可以访问或删除

💡 **提示4**: 定期清理不用的项目和备份，保持列表整洁

💡 **提示5**: 刷新按钮可以更新项目列表，查看最新状态

## 注意事项

⚠️ **注意1**: 删除操作不可恢复，请谨慎操作

⚠️ **注意2**: 确保zip文件中包含完整的项目文件

⚠️ **注意3**: 大文件上传可能需要等待，请耐心等待进度完成

⚠️ **注意4**: 项目名称不能包含 `/` `\` 等特殊字符

⚠️ **注意5**: 如遇问题，可查看浏览器控制台错误信息
